# Sprint package v0

Un `sprint_package` es el producto vendible. Un `sprint` es una instancia corrida por un equipo.

## Primer SKU sugerido

```yaml
sprint_package:
  id: marketing_30d
  name: "AI Readiness Sprint — Marketing 30 dias"
  target_buyer: marketing_director
  target_team_size: 8-50
  duration_days: 30
  included_cases: 8
  includes_resimulation: true
  includes_manager_report: true
  includes_manager_recommendation: true
  buyer_actions: [pilotar, entrenar, pausar, escalar]
  status: draft
```

## Que incluye v0

- baseline inicial
- casos vivos por rol de marketing
- beat real de IA en casos clave
- re-simulacion 7-14 dias despues en casos criticos
- evidence snapshots por persona/equipo
- reporte ejecutivo final
- recomendacion accionable para el manager

## Que no incluye v0

- casos creados libremente por la empresa
- BYO-key
- benchmarks relativos por industria
- transcript individual completo para manager
- workflow automation en herramientas del cliente
- pricing enterprise complejo

## Regla de producto

El Sprint debe poder venderse antes de que exista UI final. Si no se puede explicar como paquete cerrado, no esta listo.
